# COCoLa おでかけスポットマップ 要件整理

## 概要
COCoLaホームページ上に、千葉ニュータウンエリアの「市民おすすめスポット」を地図で見られる Web アプリを作成する。  
Google マップ上にカテゴリ別のピンを表示し、市民がフォームからスポット情報を投稿できる仕組みとする。  
既存スポット情報の修正も可能にする。  
将来的には、クリエイターズマップや文化財マップと同じ思想・共通基盤で展開する構想。

---

## 目的
- 市民おすすめスポットを地域内で共有する
- 千葉ニュータウンエリアの回遊性を高める
- 地域の魅力を見える化する
- COCoLaの地域情報基盤として活用する
- 将来的に他テーマのマップと横展開できるようにする

---

## 想定利用者
### 1. 閲覧者
- 地図を閲覧する一般ユーザー
- スポット情報を確認する

### 2. 投稿者
- フォームから新規スポットを投稿する市民
- 既存スポットの修正情報を送る

### 3. 管理者
- 投稿内容を確認する
- 公開可否を判断する
- 修正内容を反映する
- 不適切投稿や重複投稿を整理する

---

## 基本機能
- Google マップ上にスポットを表示する
- カテゴリ別にピンを切り替えて表示できる
- 市民がフォームからスポット情報を登録できる
- 既存スポット情報の修正ができる
- 千葉ニュータウンエリアの境界を明確に表示する
- 将来的に他マップ（クリエイターズマップ、文化財マップ等）と共通化しやすい構造にする
- おすすめ写真をカルーセル形式で閲覧できる

---

## 地図表示機能
- Google マップを利用する
- スポットをピンで表示する
- カテゴリ別に表示・非表示を切り替えられるようにする
- 千葉ニュータウンエリアの境界を明確化して表示する
- 境界ポリゴンは詳細データへ差し替え可能とし、必要に応じて複数ポリゴンも扱える構造にする

### カテゴリ例
- ショッピング
- 公園
- ぐるめ
- その他

※ 最終カテゴリ一覧は今後確定する

### カテゴリ運用方針
- 初期カテゴリは設定ファイルで定義する
- フォーム入力時に新しいカテゴリが任意追加されても表示できるようにする
- 新カテゴリはデータ側のカテゴリ名から自動的に表示フィルタへ反映できる構造とする

## 投稿機能
市民が簡単なフォームからスポット情報を登録できるようにする。

### 投稿項目（たたき台）
- スポット名
- カテゴリ
- 説明文
- 住所
- おすすめ理由
- 投稿者名またはニックネーム
- 連絡先（必要に応じて非公開保持）
- URL または SNS
- 写真（必要なら）

### 投稿時のカテゴリ仕様
- 既存カテゴリから選択できるようにする
- 必要に応じて投稿者が新しいカテゴリ名を任意入力できるようにする
- 管理者確認後、追加カテゴリもそのまま表示機能へ反映できる構造にする

※ 入力項目の最終確定は今後行う

---

## 修正機能
- 既存スポット情報に対して修正を行えるようにする
- 修正時には再ジオコーディングを行う
- 同じ住所に対して毎回 API を呼ばない
- 一度変換した座標は保存し、再利用する

---

## 公開・運用ルール
### 前提
市民参加型のため、誤情報・重複・不適切投稿への対応が必要。

### 想定ルール
- 投稿内容は管理者確認後に公開する想定
- 修正内容も管理者確認後に反映する想定
- 不適切な投稿は管理者が非公開・削除できる
- 重複スポットは管理者が統合・整理する

※ 承認フローは今後正式確定する

## 技術構成（現時点の想定）
### 利用するAPI
- Maps JavaScript API
- Geocoding API

### 利用しないAPI
- Places API
- 候補補完機能
- 経路案内系 API

### 理由
- 必要機能を最小構成で実現するため
- コストを抑えるため
- 構造をシンプルに保つため

---

## Google Maps API 利用条件
### 想定利用規模
- 月間閲覧数：500
- 月間新規投稿数：10件
- 月間修正件数：10件

### ジオコーディング仕様
- 新規投稿時に住所から緯度経度へ変換する
- 修正時にも再ジオコーディングする
- 同じ住所に対して毎回 API を叩かない
- 一度変換した座標は保存して再利用する
- 閲覧時には再ジオコーディングしない

### コスト感（概算）
- 高コスト構成ではない
- ざっくり月 0円〜数百円台程度の可能性が高い
- ただし、正式な金額は最新の Google 公式料金表確認が必要

※ この金額は厳密見積りではなく概算前提

---

## データ処理方針
- 投稿時または修正時に住所を緯度経度へ変換する
- 変換済み座標を保存する
- 地図表示時は保存済み座標を使ってピン表示する
- 表示のたびに住所変換はしない
- 境界データは詳細なポリゴン座標を JSON で保持する
- 写真データはスポットごとに複数枚保持できるようにする
- カテゴリ定義外の値がデータに含まれても表示側で吸収する
